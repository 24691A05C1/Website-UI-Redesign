Step 01

Heuristic Evaluation

Nielsen's 10 usability heuristics applied — surfacing friction points, inconsistencies, and accessibility gaps across the original interface.

#01
● High Severity
No Visible System Status
Loading states, form submission feedback, and async actions lacked any indicator, leaving users uncertain whether their actions had any effect.
#02
● High Severity
Inconsistent Navigation Labels
Menu items used different terminology across pages (e.g., "Account" vs "Profile" vs "My Settings"), violating the consistency heuristic.
#03
◐ Medium Severity
Poor Error Message Quality
Generic "Something went wrong" errors gave users no context or recovery path, hindering error recovery and increasing support burden.
#04
● High Severity
Insufficient Colour Contrast
Body text at 3.8:1 ratio failed WCAG AA. CTA buttons at 2.1:1 were below minimum thresholds, making them illegible for low-vision users.
#05
◐ Medium Severity
No Recognition Shortcuts
Power users had no keyboard shortcuts, recent history, or quick-access patterns — forcing the same multi-click journeys on every return visit.
#06
○ Low Severity
Cluttered Visual Hierarchy
Equal visual weight applied to primary and secondary content created cognitive overload. Users struggled to determine where to look first on landing pages.
